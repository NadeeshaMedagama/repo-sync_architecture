SOLID Principles Applied
S

Single Responsibility

Each class has one purpose — GitHubClient handles GitHub API, DocumentProcessor handles chunking, MilvusVectorStore handles vector ops.

O

Open / Closed

Extend via new interface implementations (e.g., add PineconeStore) without modifying existing code.

L

Liskov Substitution

Any IVectorStore implementation can be swapped seamlessly — MilvusVectorStore or a future PineconeStore.

I

Interface Segregation

Nine small, focused interfaces (IRepositoryDiscovery, IChangeDetector, etc.) instead of one monolithic interface.

D

Dependency Inversion

UpdateOrchestrator depends on abstractions (interfaces), not concrete classes. DIContainer handles wiring.
